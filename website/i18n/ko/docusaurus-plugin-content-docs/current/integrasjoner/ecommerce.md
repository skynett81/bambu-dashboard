# 전자상거래

전자상거래 모듈은 고객, 주문 및 청구를 관리하는 완전한 시스템을 제공합니다 — 전문적 또는 세미 프로로 3D 프린트를 판매하는 분들에게 완벽합니다.

이동: **https://localhost:3443/#orders**

:::danger 전자상거래 라이선스 필요
전자상거래 모듈은 유효한 라이선스가 필요합니다. 라이선스는 **[geektech.no](https://geektech.no)를 통해서만 구매**할 수 있습니다. 활성 라이선스 없이는 모듈이 잠겨 사용할 수 없습니다.
:::

## 라이선스 — 구매 및 활성화

### 라이선스 구매

1. **[geektech.no](https://geektech.no)**에 접속하여 계정 생성
2. **Bambu Dashboard — 전자상거래 라이선스** 선택
3. 라이선스 유형 선택:

| 라이선스 유형 | 설명 | 프린터 수 |
|---|---|---|
| **Hobby** | 프린터 1대, 개인 사용 및 소규모 판매 | 1 |
| **Professional** | 최대 5대, 상업적 사용 | 1–5 |
| **Enterprise** | 무제한 프린터, 완전 지원 | 무제한 |

4. 결제 완료
5. 이메일로 **라이선스 키** 수신

### 라이선스 활성화

1. 대시보드에서 **설정 → 전자상거래**로 이동합니다
2. 다음 필드를 입력합니다:

| 필드 | 설명 | 필수 |
|------|------|------|
| **라이선스 키** | geektech.no에서 받은 32자리 16진수 키 | ✅ 예 |
| **이메일 주소** | 구매 시 사용한 이메일 | ✅ 예 |
| **도메인** | 대시보드가 실행되는 도메인 (https:// 제외) | 권장 |
| **전화번호** | 연락처 전화번호 (국가코드 포함, 예: +82) | 선택 |

### 라이선스 유형 — 식별자 바인딩

geektech.no는 라이선스를 하나 이상의 식별자에 바인딩합니다:

| 유형 | 검증 대상 | 사용 사례 |
|------|----------|----------|
| **도메인** | 도메인 이름 (예: `dashboard.company.kr`) | 고유 도메인을 가진 고정 서버 |
| **IP** | 공개 IP 주소 | 도메인 없음, 고정 IP 서버 |
| **MAC** | 네트워크 카드의 MAC 주소 | 하드웨어 바인딩 |
| **IP + MAC** | IP와 MAC 모두 일치해야 함 | 최고 보안 |

:::info 자동 식별
대시보드는 각 검증 시 서버의 IP 주소와 MAC 주소를 자동으로 전송합니다. 수동으로 입력할 필요가 없습니다 — geektech.no가 첫 번째 활성화 시 등록합니다.
:::

여러 IP 주소와 MAC 주소를 허용할 수 있습니다 (geektech.no 관리에서 줄당 하나). 여러 네트워크 카드나 동적 IP를 가진 서버에 유용합니다.

3. **라이선스 활성화** 클릭
4. 대시보드가 geektech.no에 활성화 요청을 보냅니다
5. 성공적으로 활성화되면 다음이 표시됩니다:
   - **라이선스 유형** (Hobby / Professional / Enterprise)
   - **만료일**
   - **최대 프린터 수**
   - **라이선스 보유자**
   - **인스턴스 ID** (설치에 고유)

:::warning 라이선스 키는 도메인과 설치에 연결됩니다
키는 특정 Bambu Dashboard 설치와 도메인에 활성화됩니다. 다음이 필요한 경우 [geektech.no](https://geektech.no) 지원팀에 문의하십시오:
- 새 서버로 라이선스 이전
- 도메인 변경
- 프린터 수 증가
:::

### 라이선스 유효성 검사

라이선스는 geektech.no와 인증 및 동기화됩니다:

- **시작 시 검증** — 라이선스가 자동으로 확인됩니다
- **지속적 검증** — geektech.no에 대해 24시간마다 재검증
- **오프라인 모드** — 네트워크 오류 시 캐시된 검증으로 라이선스는 최대 **7일** 작동합니다
- **만료된 라이선스** → 모듈이 잠기지만 기존 데이터(주문, 고객)는 유지됩니다
- **PIN 코드** — geektech.no는 PIN 시스템을 통해 라이선스를 잠금/해제할 수 있습니다
- **갱신** — **[geektech.no](https://geektech.no)** → 내 라이선스 → 갱신에서 진행

### 라이선스 유형 및 제한

| 플랜 | 프린터 | 플랫폼 | 수수료 | 가격 |
|------|--------|--------|--------|------|
| **Hobby** | 1 | 1 (Shopify 또는 WooCommerce) | 5% | geektech.no 참조 |
| **Professional** | 1–5 | 전체 | 5% | geektech.no 참조 |
| **Enterprise** | 무제한 | 전체 + API | 3% | geektech.no 참조 |

## 고객

### 고객 만들기

1. **전자상거래 → 고객**으로 이동합니다
2. **새 고객** 클릭
3. 다음을 입력합니다:
   - **이름 / 회사명**
   - **담당자** (기업의 경우)
   - **이메일 주소**
   - **전화번호**
   - **주소** (청구 주소)
   - **사업자등록번호** (선택 사항, 부가세 등록 업체)
   - **메모** — 내부 메모
4. **만들기** 클릭

### 고객 개요

고객 목록에 표시되는 정보:
- 이름 및 연락처 정보
- 총 주문 수
- 총 매출
- 최근 주문 날짜
- 상태 (활성 / 비활성)

고객을 클릭하면 모든 주문 및 청구 기록을 볼 수 있습니다.

## 주문 관리

### 주문 만들기

1. **전자상거래 → 주문**으로 이동합니다
2. **새 주문** 클릭
3. 목록에서 **고객** 선택
4. 주문 항목 추가:
   - 파일 라이브러리에서 파일/모델 선택하거나 자유 텍스트 항목 추가
   - 수량 및 단가 입력
   - 프로젝트에 연결된 경우 시스템이 자동으로 비용 계산
5. **납품 날짜** 입력 (예상)
6. **주문 만들기** 클릭

### 주문 상태

| 상태 | 설명 |
|---|---|
| 문의 | 문의 수신, 미확인 |
| 확인됨 | 고객이 확인 |
| 제작 중 | 프린팅 진행 중 |
| 납품 준비 | 완료, 수령/배송 대기 |
| 납품됨 | 주문 완료 |
| 취소됨 | 고객 또는 판매자가 취소 |

주문을 클릭한 후 **상태 변경**으로 상태를 업데이트합니다.

### 프린트를 주문에 연결

1. 주문 열기
2. **프린트 연결** 클릭
3. 기록에서 프린트 선택 (다중 선택 지원)
4. 프린트 기록에서 비용 데이터가 자동으로 가져와집니다

## 청구

청구에 대한 자세한 문서는 [프로젝트 → 청구](../funksjoner/projects#fakturering)를 참조하세요.

주문에서 직접 인보이스를 생성할 수 있습니다:

1. 주문 열기
2. **인보이스 생성** 클릭
3. 금액 및 부가세 확인
4. PDF 다운로드 또는 고객 이메일로 전송

### 인보이스 번호 시리즈

**설정 → 전자상거래**에서 인보이스 번호 시리즈 설정:
- **접두사**: 예: `2026-`
- **시작 번호**: 예: `1001`
- 인보이스 번호는 오름차순으로 자동 할당

## 보고 및 수수료

### 수수료 보고

시스템이 모든 거래 수수료를 추적합니다:
- **전자상거래 → 수수료**에서 수수료 확인
- 회계 목적으로 수수료를 보고됨으로 표시
- 기간별 수수료 요약 내보내기

### 통계

**전자상거래 → 통계**에서:
- 월별 매출 (막대 그래프)
- 매출 기준 상위 고객
- 가장 많이 판매된 모델/재료
- 평균 주문 규모

회계 시스템을 위해 CSV로 내보내기.
