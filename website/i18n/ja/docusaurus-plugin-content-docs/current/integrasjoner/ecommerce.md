# Eコマース

Eコマースモジュールは、顧客、注文、請求書を管理するための完全なシステムを提供します — 3Dプリントをプロフェッショナルまたはセミプロとして販売する方に最適です。

アクセス先：**https://localhost:3443/#orders**

:::danger Eコマースライセンスが必要です
Eコマースモジュールには有効なライセンスが必要です。ライセンスは**[geektech.no](https://geektech.no)からのみ購入**できます。有効なライセンスがない場合、モジュールはロックされてアクセスできません。
:::

## ライセンス — 購入とアクティベーション

### ライセンスの購入

1. **[geektech.no](https://geektech.no)**にアクセスしてアカウントを作成します
2. **Bambu Dashboard — Eコマースライセンス**を選択します
3. ライセンスタイプを選択します：

| ライセンスタイプ | 説明 | プリンター数 |
|---|---|---|
| **ホビー** | 1台のプリンター、個人使用と小規模販売 | 1 |
| **プロフェッショナル** | 最大5台のプリンター、商業利用 | 1〜5 |
| **エンタープライズ** | 無制限のプリンター、フルサポート | 無制限 |

4. 支払いを完了します
5. メールで**ライセンスキー**を受け取ります

### ライセンスのアクティベーション

1. ダッシュボードの**設定 → Eコマース**に移動します
2. 以下のフィールドを入力します：

| フィールド | 説明 | 必須 |
|-----------|------|------|
| **ライセンスキー** | geektech.noからの32文字の16進数キー | ✅ はい |
| **メールアドレス** | 購入時に使用したメール | ✅ はい |
| **ドメイン** | ダッシュボードが動作するドメイン（https://なし） | 推奨 |
| **電話番号** | 連絡先電話番号（国番号付き、例：+81） | 任意 |

### ライセンスタイプ — 識別子バインディング

geektech.noはライセンスを1つ以上の識別子にバインドします：

| タイプ | 検証対象 | 用途 |
|--------|---------|------|
| **ドメイン** | ドメイン名（例：`dashboard.company.jp`） | 独自ドメインを持つ固定サーバー |
| **IP** | パブリックIPアドレス | ドメインなし、固定IPのサーバー |
| **MAC** | ネットワークカードのMACアドレス | ハードウェアバインディング |
| **IP + MAC** | IPとMACの両方が一致する必要あり | 最高セキュリティ |

:::info 自動識別
ダッシュボードは各検証時にサーバーのIPアドレスとMACアドレスを自動的に送信します。手動で入力する必要はありません — geektech.noは初回アクティベーション時に登録します。
:::

複数のIPアドレスとMACアドレスを許可できます（geektech.noの管理画面で1行ずつ）。複数のネットワークカードや動的IPを持つサーバーに便利です。

3. **ライセンスをアクティベート**をクリックします
4. ダッシュボードがgeektech.noにアクティベーションリクエストを送信します
5. アクティベーション成功時に以下が表示されます：
   - **ライセンスタイプ**（ホビー / プロフェッショナル / エンタープライズ）
   - **有効期限**
   - **最大プリンター数**
   - **ライセンスホルダー**
   - **インスタンスID**（インストール固有）

:::warning ライセンスキーはドメインとインストールに紐付けられます
キーは特定のBambu Dashboardインストールとドメインに対してアクティベートされます。以下が必要な場合は[geektech.no](https://geektech.no)サポートにお問い合わせください：
- ライセンスを新しいサーバーに移動
- ドメインの変更
- プリンター数の増加
:::

### ライセンス検証

ライセンスはgeektech.noで認証および同期されます：

- **起動時の検証** — ライセンスは自動的にチェックされます
- **継続的な検証** — geektech.noに対して24時間ごとに再検証
- **オフラインモード** — ネットワーク障害時、キャッシュされた検証でライセンスは最大**7日間**機能します
- **ライセンス期限切れ** → モジュールがロックされますが、既存データ（注文、顧客）は保持されます
- **PINコード** — geektech.noはPINシステムでライセンスをロック/解除できます
- **更新** — **[geektech.no](https://geektech.no)** → マイライセンス → 更新から

### ライセンスタイプと制限

| プラン | プリンター | プラットフォーム | 手数料 | 価格 |
|--------|-----------|----------------|--------|------|
| **ホビー** | 1 | 1（Shopify または WooCommerce） | 5% | geektech.noを参照 |
| **プロフェッショナル** | 1〜5 | すべて | 5% | geektech.noを参照 |
| **エンタープライズ** | 無制限 | すべて + API | 3% | geektech.noを参照 |

## 顧客

### 顧客の作成

1. **Eコマース → 顧客**に移動します
2. **新しい顧客**をクリックします
3. 以下を入力します：
   - **名前 / 会社名**
   - **担当者**（法人の場合）
   - **メールアドレス**
   - **電話番号**
   - **住所**（請求先住所）
   - **法人番号 / 個人番号**（オプション、消費税登録事業者向け）
   - **メモ** — 内部メモ
4. **作成**をクリックします

### 顧客一覧

顧客リストには以下が表示されます：
- 名前と連絡先情報
- 総注文数
- 総売上高
- 最終注文日
- ステータス（アクティブ / 非アクティブ）

顧客をクリックして全注文と請求履歴を表示します。

## 注文管理

### 注文の作成

1. **Eコマース → 注文**に移動します
2. **新しい注文**をクリックします
3. リストから**顧客**を選択します
4. 注文明細を追加します：
   - ファイルライブラリからファイル/モデルを選択するか、フリーテキスト項目を追加
   - 数量と単価を入力
   - プロジェクトにリンクするとシステムがコストを自動計算
5. **納期**（推定）を設定します
6. **注文を作成**をクリックします

### 注文ステータス

| ステータス | 説明 |
|---|---|
| 問い合わせ | リクエスト受信、未確認 |
| 確認済み | 顧客が確認 |
| 製造中 | プリント進行中 |
| 配送準備完了 | 完成、受け取り/発送待ち |
| 配送済み | 注文完了 |
| キャンセル | 顧客またはあなたによりキャンセル |

注文をクリックして**ステータスを変更**でステータスを更新します。

### プリントを注文にリンク

1. 注文を開きます
2. **プリントをリンク**をクリックします
3. 履歴からプリントを選択します（複数選択可）
4. コストデータがプリント履歴から自動的に取得されます

## 請求書

詳細な請求書ドキュメントについては[プロジェクト → 請求書](../funksjoner/projects#fakturering)を参照してください。

注文から請求書を直接生成できます：

1. 注文を開きます
2. **請求書を生成**をクリックします
3. 金額と消費税を確認します
4. PDFをダウンロードするか顧客のメールに送信します

### 請求書番号シリーズ

**設定 → Eコマース**で請求書番号シリーズを設定します：
- **プレフィックス**：例 `2026-`
- **開始番号**：例 `1001`
- 請求書番号は昇順で自動付番されます

## レポートと手数料

### 手数料レポート

システムはすべての取引手数料を追跡します：
- **Eコマース → 手数料**で手数料を表示
- 会計目的で手数料を報告済みとしてマーク
- 期間ごとの手数料概要をエクスポート

### 統計

**Eコマース → 統計**で以下を確認：
- 月次売上高（棒グラフ）
- 売上高上位顧客
- 最も売れているモデル/素材
- 平均注文サイズ

会計システム向けにCSVにエクスポートします。
