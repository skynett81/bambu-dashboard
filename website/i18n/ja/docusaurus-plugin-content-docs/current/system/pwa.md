# PWA（Progressive Web App）

Bambu DashboardはProgressive Web App（PWA）としてインストールできます — アプリストアなしにブラウザから直接アプリライクなエクスペリエンスを提供します。素早いアクセス、バックグラウンドプッシュ通知、限定的なオフライン機能を利用できます。

## アプリとしてインストール

### デスクトップ（Chrome / Edge / Chromium）

1. ブラウザで `https://localhost:3443` を開きます
2. アドレスバーの**インストール**アイコン（画面アイコン付きの下矢印）を確認します
3. クリックします
4. ダイアログで**インストール**をクリックします
5. Bambu DashboardがブラウザUIなしの独自のウィンドウとして開きます

または：3点メニュー（⋮）をクリックして**Bambu Dashboardをインストール...**を選択します。

### デスクトップ（Firefox）

FirefoxはフルPWAインストールを直接サポートしていません。最良のエクスペリエンスにはChromeまたはEdgeを使用してください。

### モバイル（Android – Chrome）

1. Chromeで**https://あなたのサーバーIP:3443**を開きます
2. 3点メニューをタップして**ホーム画面に追加**を選択します
3. アプリに名前を付けて**追加**をタップします
4. アイコンがホーム画面に表示されます — アプリがブラウザUIなしのフルスクリーンで開きます

### モバイル（iOS – Safari）

1. Safariで**https://あなたのサーバーIP:3443**を開きます
2. **共有**アイコン（上矢印付きの四角）をタップします
3. 下にスクロールして**ホーム画面に追加**を選択します
4. **追加**をタップします

:::warning iOSの制限
iOSのPWAサポートは限定的です。プッシュ通知はiOS 16.4以降でのみ機能します。オフラインモードは制限されています。
:::

## オフラインモード

PWAは限定的なオフライン使用に必要なリソースをキャッシュします。

| 機能 | オフライン利用可能 |
|---|---|
| 最後に確認されたプリンターステータス | ✅（キャッシュから） |
| プリント履歴 | ✅（キャッシュから） |
| フィラメント在庫 | ✅（キャッシュから） |
| リアルタイムステータス（MQTT） | ❌ 接続が必要 |
| カメラストリーム | ❌ 接続が必要 |
| プリンターへのコマンド送信 | ❌ 接続が必要 |

オフライン表示では上部にバナーが表示されます：「接続が失われました — 最後に確認されたデータを表示しています」。

## バックグラウンドプッシュ通知

PWAはアプリが開いていなくてもプッシュ通知を送信できます。

1. PWAを開きます
2. **設定 → 通知 → ブラウザプッシュ**に移動します
3. **プッシュ通知を有効にする**をクリックします
4. 権限ダイアログを承認します
5. 通知がOSの通知センターに配信されます

プッシュ通知は[通知](../funksjoner/notifications)で設定されたすべてのイベントに対して機能します。

:::info Service Worker
プッシュ通知にはブラウザがバックグラウンドで実行されていることが必要です（完全なシステムシャットダウンなし）。PWAは受信にService Workerを使用します。
:::

## アプリアイコンと外観

PWAはBambu Dashboardアイコンを自動的に使用します。カスタマイズするには：

1. **設定 → システム → PWA**に移動します
2. カスタムアイコンをアップロードします（最低512×512 px PNG）
3. **アプリ名**と**短い名前**（モバイルでアイコンの下に表示）を設定します
4. モバイルのステータスバー用の**テーマカラー**を選択します

## PWAの更新

サーバーが更新されると、PWAは自動的に更新されます。

- 控えめなバナーが表示されます：「新しいバージョンが利用可能 — クリックして更新」
- バナーをクリックして新しいバージョンを読み込みます
- 手動での再インストールは不要です
