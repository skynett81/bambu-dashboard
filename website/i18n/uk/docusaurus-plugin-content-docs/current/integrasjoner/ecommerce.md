# Електронна торгівля

Модуль електронної торгівлі надає вам повну систему для управління клієнтами, замовленнями та виставленням рахунків — ідеально для тих, хто продає 3D-друки на професійному або напівпрофесійному рівні.

Перейдіть до: **https://localhost:3443/#orders**

:::danger Потрібна ліцензія електронної торгівлі
Модуль електронної торгівлі вимагає дійсної ліцензії. Ліцензії можна **придбати виключно на [geektech.no](https://geektech.no)**. Без активної ліцензії модуль заблокований та недоступний.
:::

## Ліцензія — придбання та активація

### Придбання ліцензії

1. Перейдіть на **[geektech.no](https://geektech.no)** та створіть акаунт
2. Виберіть **Bambu Dashboard — Ліцензія електронної торгівлі**
3. Виберіть тип ліцензії:

| Тип ліцензії | Опис | Принтери |
|---|---|---|
| **Хобі** | Один принтер, особисте використання та невеликі продажі | 1 |
| **Професійна** | До 5 принтерів, комерційне використання | 1–5 |
| **Enterprise** | Необмежена кількість принтерів, повна підтримка | Необмежено |

4. Завершіть оплату
5. Ви отримаєте **ліцензійний ключ** електронною поштою

### Активація ліцензії

1. Перейдіть до **Налаштування → Електронна торгівля** в дашборді
2. Заповніть наступні поля:

| Поле | Опис | Обов'язково |
|------|------|------------|
| **Ліцензійний ключ** | 32-символьний шістнадцятковий ключ від geektech.no | ✅ Так |
| **Електронна пошта** | Пошта, використана при купівлі | ✅ Так |
| **Домен** | Домен, на якому працює дашборд (без https://) | Рекомендовано |
| **Телефон** | Контактний телефон (з кодом країни, напр. +380) | Необов'язково |

### Тип ліцензії — прив'язка ідентифікатора

geektech.no прив'язує ліцензію до одного або кількох ідентифікаторів:

| Тип | Перевіряє проти | Сценарій використання |
|-----|----------------|-----------------------|
| **Домен** | Доменне ім'я (напр. `dashboard.company.ua`) | Фіксований сервер з власним доменом |
| **IP** | Публічна IP-адреса(и) | Сервер без домену, фіксований IP |
| **MAC** | MAC-адреса(и) мережевої карти | Апаратне прив'язування |
| **IP + MAC** | IP та MAC повинні збігатися | Найвищий рівень безпеки |

:::info Автоматична ідентифікація
Дашборд автоматично надсилає IP-адресу та MAC-адресу сервера при кожній перевірці. Не потрібно вводити їх вручну — geektech.no реєструє їх при першій активації.
:::

Можна дозволити кілька IP-адрес і MAC-адрес (по одному рядку в адмін-панелі geektech.no). Це корисно для серверів з кількома мережевими картами або динамічним IP.

3. Натисніть **Активувати ліцензію**
4. Дашборд надсилає запит на активацію до geektech.no
5. При успішній активації відображається:
   - **Тип ліцензії** (Хобі / Професійна / Enterprise)
   - **Термін дії**
   - **Максимальна кількість принтерів**
   - **Власник ліцензії**
   - **ID екземпляра** (унікальний для вашої інсталяції)

:::warning Ліцензійний ключ прив'язаний до вашого домену та інсталяції
Ключ активується для конкретної інсталяції Bambu Dashboard та домену. Зверніться до підтримки [geektech.no](https://geektech.no), якщо вам потрібно:
- Перенести ліцензію на новий сервер
- Змінити домен
- Збільшити кількість принтерів
:::

### Перевірка ліцензії

Ліцензія аутентифікується та синхронізується з geektech.no:

- **Перевірка при запуску** — ліцензія перевіряється автоматично
- **Постійна перевірка** — переперевіряється кожні 24 години в geektech.no
- **Офлайн-режим** — при відсутності мережевого з'єднання ліцензія діє до **7 днів** з кешованою перевіркою
- **Прострочена ліцензія** → модуль блокується, але існуючі дані (замовлення, клієнти) зберігаються
- **PIN-код** — geektech.no може заблокувати/розблокувати ліцензію через систему PIN
- **Продовження** — через **[geektech.no](https://geektech.no)** → Мої ліцензії → Продовжити

### Типи ліцензій та обмеження

| Тарифний план | Принтери | Платформи | Комісія | Ціна |
|---------------|----------|-----------|---------|------|
| **Хобі** | 1 | 1 (Shopify АБО WooCommerce) | 5% | Дивіться geektech.no |
| **Професійна** | 1–5 | Усі | 5% | Дивіться geektech.no |
| **Enterprise** | Необмежено | Усі + API | 3% | Дивіться geektech.no |

## Клієнти

### Створення клієнта

1. Перейдіть до **Електронна торгівля → Клієнти**
2. Натисніть **Новий клієнт**
3. Заповніть:
   - **Ім'я / Назва компанії**
   - **Контактна особа** (для компаній)
   - **Електронна пошта**
   - **Телефон**
   - **Адреса** (платіжна адреса)
   - **ІПН / Код ЄДРПОУ** (необов'язково, для платників ПДВ)
   - **Примітка** — внутрішня позначка
4. Натисніть **Створити**

### Огляд клієнтів

Список клієнтів показує:
- Ім'я та контактну інформацію
- Загальну кількість замовлень
- Загальний оборот
- Дату останнього замовлення
- Статус (Активний / Неактивний)

Натисніть на клієнта для перегляду всієї історії замовлень та рахунків.

## Управління замовленнями

### Створення замовлення

1. Перейдіть до **Електронна торгівля → Замовлення**
2. Натисніть **Нове замовлення**
3. Виберіть **Клієнта** зі списку
4. Додайте рядки замовлення:
   - Виберіть файл/модель з бібліотеки файлів, або додайте довільний рядок
   - Вкажіть кількість та ціну за одиницю
   - Система розраховує вартість автоматично при прив'язці до проекту
5. Вкажіть **Дату доставки** (орієнтовну)
6. Натисніть **Створити замовлення**

### Статус замовлення

| Статус | Опис |
|---|---|
| Запит | Отримано запит, не підтверджено |
| Підтверджено | Клієнт підтвердив |
| У виробництві | Друк виконується |
| Готово до доставки | Готово, очікує на отримання/відправлення |
| Доставлено | Замовлення виконано |
| Скасовано | Скасовано клієнтом або вами |

Оновіть статус, натиснувши на замовлення → **Змінити статус**.

### Прив'язка друків до замовлення

1. Відкрийте замовлення
2. Натисніть **Прив'язати друк**
3. Виберіть друки з історії (множинний вибір підтримується)
4. Дані витрат автоматично беруться з історії друку

## Виставлення рахунків

Дивіться [Проекти → Виставлення рахунків](../funksjoner/projects#fakturering) для детальної документації.

Рахунок-фактура може бути згенерований безпосередньо із замовлення:

1. Відкрийте замовлення
2. Натисніть **Створити рахунок**
3. Перевірте суму та ПДВ
4. Завантажте PDF або надішліть на електронну пошту клієнта

### Нумерація рахунків

Налаштуйте серію номерів рахунків у **Налаштування → Електронна торгівля**:
- **Префікс**: наприклад `2026-`
- **Початковий номер**: наприклад `1001`
- Номери рахунків призначаються автоматично у порядку зростання

## Звітність та комісії

### Звітність про комісії

Система відстежує всі транзакційні комісії:
- Дивіться комісії в **Електронна торгівля → Комісії**
- Позначайте комісії як звітні для цілей бухгалтерії
- Експортуйте зведення комісій за період

### Статистика

У **Електронна торгівля → Статистика**:
- Місячний оборот (стовпчаста діаграма)
- Топ-клієнти за оборотом
- Найбільш продавані моделі/матеріали
- Середній розмір замовлення

Експортуйте у CSV для бухгалтерської системи.
